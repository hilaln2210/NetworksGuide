מה זה אבטחת מידע בכלל? — מבוא מאפס
לגמרי מהתחלה · ללא ידע מוקדם · למה זה רלוונטי לכולם
התחלה⏱ 20m▶
לפני שנדבר על CIA Triad, Firewalls ו-Penetration Testing — שאלה אחת: למה אבטחת מידע רלוונטית לך, גם אם אתה לא האקר?

כל אפליקציה שאתה בונה, כל DB שאתה מנהל, כל API שאתה חושף — יש מישהו שרוצה לנצל אותם. הפרק הזה מסביר את הבסיס מאפס.
מה זה בכלל "אבטחת מידע"?
🏦 הדמיון לבנק:
בנק מגן על הכסף שלך בכמה שכבות: דלת פלדה, מצלמות, שומרים, קוד סודי, חתימה.
אם אחת השכבות נפרצת — יש עוד שכבות שמגנות.

אבטחת מידע = אותו עיקרון, על נתונים דיגיטליים. מגן על מידע (Data) מפני גישה לא מורשית, שינוי, או מניעת שירות.
מי מנסה לפרוץ ולמה?
💰
תוקפים כלכליים: Ransomware, גניבת כרטיסי אשראי, מכירת מידע ב-Dark Web. הכי נפוץ.
🏳️
מדינות (Nation-State): ריגול, גניבת קניין רוחני, שיבוש תשתיות. SolarWinds, Colonial Pipeline.
😤
Hacktivists: פעולה פוליטית/אידיאולוגית — DDoS על אתרי ממשלה, דלף מסמכים.
🎓
Script Kiddies: מתחילים שמשתמשים בכלים מוכנים בלי להבין. רועשים אבל פחות מסוכנים.

🔺 כל אירוע אבטחה פוגע באחד מהשלושה:

דוגמה: DDoS שמפיל אתר, Ransomware שנועל קבצים

יסודות אבטחת מידע — CIA Triad
Confidentiality, Integrity, Availability, Threat Modeling, Risk
CORE⏱ 40m▶
אם אבטחת מידע היא הבניין, CIA Triad הוא הבסיס שעליו הכל עומד. לפני שנבין Firewalls, הצפנה או Penetration Testing — חייבים להבין את שלושת השאלות שכל החלטת אבטחה נשפטת לפיהן.
🔺 CIA Triad — שלושה שאלות, אינסוף תרחישים:

C — Confidentiality: "רק מי שאמור לראות — רואה."
I — Integrity: "מה שנשמר = מה שנשלח. לא שונה."
A — Availability: "המערכת זמינה כשצריך אותה."

כל פרצת אבטחה שתשמע עליה — בדוק מה מהשלושה הופר. לרוב זה Confidentiality, לפעמים כולם ביחד.

Risk Management — Likelihood × Impact, Risk Appetite
Security Controls — Preventive, Detective, Corrective, Deterrent

כשל אגדי — Equifax 2017
🔴 BREACH — 2017 / Equifax
143 מיליון אמריקאים נפגעו בגלל Patch שלא הותקן
ביולי 2017 גילתה Equifax שתוקפים שהו ברשת שלה 76 ימים — בשקט, ללא גילוי. הם גנבו שמות, מספרי ביטוח לאומי, תאריכי לידה ומספרי כרטיסי אשראי של 143 מיליון אמריקאים.

הכניסה: CVE-2017-5638 ב-Apache Struts — Patch היה זמין חודשיים קודם. Equifax לא התקינה. הופרו כל שלושת עמודי ה-CIA. עלות: 700 מיליון דולר.
💡 Least Privilege בפרקטיקה: כל משתמש, שירות, ו-API Key — מקבל רק את מה שהוא צריך. עובד שעוזב? כל ה-Permissions נשללים מיידית. Service Account? קורא בלבד אלא אם כן חייב לכתוב.

קריפטוגרפיה — ההצפנה שמגנה על הכל
Symmetric, Asymmetric, Hashing, PKI, TLS, Certificates
CRYPTO⏱ 60m▶
כשאתה שולח הודעה, מתחבר ל-WiFi, או משתמש ב-HTTPS — קריפטוגרפיה מגנה עליך. זו הטכנולוגיה שהופכת נתונים לבלתי קריאים לכל מי שאין לו את המפתח הנכון. חשוב להבין: לא כל "הצפנה" שווה. MD5 "להצפנה" = לא בטוח. AES-256 = בטוח.
🔐 שני סוגי מנעולים:
Symmetric (סימטרי) = מנעול עם מפתח אחד: מי שנועל — הוא גם פותח. מהיר, אבל איך שולחים את המפתח בבטחה?

Asymmetric (א-סימטרי) = תיבת דואר: לכולם יש את כתובתך (Public Key) — שולחים לך, אבל רק אתה פותח (Private Key).

בפרקטיקה: Asymmetric מחליפים מפתח → Symmetric מצפין הכל. זה בדיוק מה שTLS עושה.
מה לומדים
Symmetric Encryption — AES-256, מפתח אחד, מהיר אבל בעיית החלפת מפתח
Asymmetric Encryption — RSA, ECC, Public/Private Key Pair
Hashing — SHA-256, bcrypt, argon2 — Salting ולמה MD5 מת
Digital Signatures — איך מוכיחים שהמייל אכן מגיע ממך
PKI — Certificate Authorities, TLS Handshake, Chain of Trust
Perfect Forward Secrecy — למה Ephemeral Keys חשובים
Hashing — לא הצפנה, אבל קריטי
מה?
Hash = פונקציה חד-כיוונית. "password123" → "482c81..." — לא ניתן לחזור לאחור. לכן DB שומר Hashes, לא סיסמאות.
❌ MD5
GPU מודרני: 50 מיליארד MD5 hashes בשנייה. rockyou.txt (14M סיסמאות) נפצח בשניות. MD5 מת לסיסמאות.
✅ bcrypt
bcrypt(12): 5,000 hashes בשנייה. פי 10 מיליון איטי יותר. Salt מובנה. rockyou.txt = שנים לפצח.
🥇 argon2
argon2id: מומלץ OWASP. מנצח גם GPU attacks וגם ASIC attacks. הבחירה ב-2024.
כשל אגדי — Heartbleed 2014
🔴 VULN — 2014 / OpenSSL Heartbleed
באג אחד בOpenSSL חשף Private Keys של מחצית האינטרנט
Heartbleed (CVE-2014-0160): באג בפיצ'ר "Heartbeat" — שרת הצהיר על מחרוזת ארוכה יותר ממה שהיא. הספרייה החזירה 64KB מהזיכרון ללא בדיקה.

אותם 64KB יכלו להכיל סיסמאות, Session Cookies, ואפילו Private Keys. 500,000 שרתים חשופים. הבאג חי בקוד 2 שנים.
💡 Perfect Forward Secrecy: כל Session מקבל Ephemeral Key חדש. גם אם Private Key של השרת נחשף בעתיד — שיחות עבר לא ניתנות לפיענוח. TLS 1.3 מחייב זאת.
📖 מילון מושגים — פרק זה▶
Symmetric Encryptionמפתח אחד מצפין ופותח. AES-256 הוא הסטנדרט. מהיר אבל מצריך החלפת מפתח בטוחה.
Asymmetric Encryptionזוג מפתחות: Public (לכולם) ו-Private (רק אצלך). RSA, ECC. איטי יותר.
Hashingפונקציה חד-כיוונית. Input → Hash. לא ניתן לחזור. לאחסון סיסמאות, לא להצפנה.
Saltערך אקראי שמוסיפים לסיסמה לפני Hash. מונע Rainbow Table Attacks.
TLSTransport Layer Security — פרוטוקול ההצפנה מאחורי HTTPS. מבטיח Confidentiality + Integrity.
PKIPublic Key Infrastructure — מערכת Certificate Authorities שמאמתת זהויות ב-HTTPS.
Certificate Authorityגוף מהימן שחותם על תעודות SSL. Let's Encrypt, DigiCert. "אני מאשר שזה אכן google.com."
Perfect Forward Secrecyכל Session עם Ephemeral Key. גם אם Private Key נחשף — שיחות עבר בטוחות.

אבטחת רשת — Firewalls, IDS, VPN
Network Segmentation, Zero Trust, NGFW, IDS/IPS, VPN, DNS Security
NETWORK⏱ 55m▶
רשת לא מאובטחת = כביש פתוח שכולם יכולים לנסוע עליו ולראות מה אתה מוביל. אבטחת רשת היא שכבת ההגנה שבין המשתמש לשרת — Firewalls, IDS, VPN ועוד. הפרק הזה גם מסביר למה Zero Trust ביטל את הרעיון של "רשת פנימית בטוחה".
🏰 מהמבצר לZero Trust:
מודל ישן (Perimeter): מבצר עם חומה. מבפנים — בטוח. מבחוץ — אויב. בעיה: מה אם מישהו כבר בפנים?

Zero Trust — המודל המודרני: "Never Trust, Always Verify." גם עובד פנימי, גם שרת פנימי — חייבים לאמת כל גישה, תמיד, לכל Resource.
מה לומדים
Firewall Types — Stateless, Stateful, NGFW, WAF
IDS vs IPS — גילוי מול מניעה, Signature vs Anomaly
Network Segmentation — DMZ, VLANs, Micro-segmentation
Zero Trust Network — "Never Trust, Always Verify"
VPN — IPSec, SSL/TLS VPN, Split Tunneling
DNS Security — DNS over HTTPS, DNSSEC, DNS Filtering
Firewall — מי מורשה להיכנס
Stateless
בודק כל Packet בנפרד לפי Rules. מהיר אבל לא מבין הקשר — ניתן לעקוף.
Stateful
זוכר Connections פתוחים. מבין שPacket שייך לSession קיים. הסטנדרט של היום.
NGFW
Next-Gen Firewall: Deep Packet Inspection, Application awareness, IPS מובנה, SSL Inspection.
WAF
Web Application Firewall: מגן ספציפית על HTTP — SQL Injection, XSS, OWASP Top 10.

🔴 RANSOMWARE — 2021 / Colonial Pipeline
צינור הדלק הגדול בארה"ב הושבת — סיסמה אחת מהDark Web

🔴 BREACH — 2017 / Equifax
143 Million Americans Exposed Because of an Uninstalled Patch
In July 2017, Equifax — one of the world's largest credit rating agencies — discovered that attackers had been inside their network for 76 days, silently, undetected. They stole names, Social Security numbers, dates of birth, addresses, and credit card numbers of 143 million Americans.

Famous Breach
🔴 VULN — 2014 / OpenSSL Heartbleed
One Bug in OpenSSL Exposed Passwords and Private Keys of Half the Internet
Heartbleed (CVE-2014-0160) was a vulnerability in OpenSSL — the library securing HTTPS. A simple bug in the "Heartbeat" feature (a connection keep-alive check): a server sent a short string but declared it was long. The library returned 64KB from memory — unchecked.

Those 64KB could contain plaintext passwords, session cookies, and even SSL certificate private keys. An attacker could read repeatedly and receive different memory chunks each time. The bug lived in the code for two years before discovery. 500,000 internet servers were exposed. Lesson: Code Audits, Fuzzing, and Memory-Safe Languages (Rust).

Famous Breach
🔴 MISCONFIGURATION — 2019 / Capital One
100 Million Customers Exposed — SSRF + Misconfigured IAM Role
In 2019, the Capital One breach exposed data of 100 million US and Canadian customers. The attacker, Paige Thompson, a former AWS employee, exploited SSRF (Server-Side Request Forgery) in Capital One's Web Application Firewall.

She sent a request that caused the server to query the AWS Metadata Service and retrieve IAM Role credentials. That Role, due to misconfiguration, had overly broad permissions to S3 Buckets containing customer data. 03
rockyou.txt: A file from 2009 containing 14 million real passwords from the RockYou breach. "password123", "123456", "qwerty" — all in there. Dictionary Attack checks every password on the list. MD5 of "password" is cracked in under a second.

06
Salt protects against Rainbow Tables: Rainbow Table = a pre-computed lookup of MD5("password")→Hash. When adding a random Salt: MD5("password"+Salt) → Hash unique per user. Rainbow Table doesn't help because it didn't precompute this Salt. bcrypt has Salt built in.
