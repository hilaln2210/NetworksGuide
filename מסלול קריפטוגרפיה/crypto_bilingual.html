יסודות — Encryption vs Hashing vs Encoding
ההבדל בין קידוד, הצפנה וגיבוב — הבסיס לכל מה שבא אחרי
CORE20m▶
שלושה מושגים שכולם מבלבלים: Encoding, Encryption, Hashing. הם נשמעים דומה אבל עושים דברים שונים לגמרי. אם לא תבין את ההבדל — תבנה מערכות שבורות.
📬Encoding = לתרגם לשפה אחרת (כולם יכולים לקרוא). Encryption = לנעול בכספת (רק מי שיש לו מפתח). Hashing = טביעת אצבע (חד-כיוונית, בלתי הפיכה).
מה לומדים
Encoding (Base64, Hex) — שינוי פורמט, לא הגנה
Encryption (AES, RSA) — הצפנה עם מפתח סודי
Hashing (SHA-256, MD5) — פונקציה חד-כיוונית
סימטרי vs אסימטרי — מפתח אחד או זוג
Public/Private keys — הבסיס של אבטחה מודרנית
חתימות דיגיטליות — איך מוכיחים זהות
01
Encoding: Base64 הופך בינארי לטקסט. Hello → SGVsbG8=. כל אחד יכול לפענח — זה לא הגנה.
02
Symmetric Encryption: מפתח אחד מצפין ומפענח. AES-256 — הסטנדרט. כמו כספת עם מפתח אחד.
03
Asymmetric Encryption: שני מפתחות — Public (מצפין) ו-Private (מפענח). RSA, ECC. כמו תיבת דואר — כל אחד יכול להכניס, רק אתה פותח.
04
Hashing: SHA-256 הופך כל input ל-256 ביט. חד-כיווני — אי אפשר לחזור. שינוי של ביט אחד → hash שונה לגמרי.
05
Digital Signatures: חותם עם Private Key, מאמת עם Public Key. מוכיח שההודעה לא שונתה ושהשולח אמיתי.
💡דוגמה אמיתית: LinkedIn breach 2012 — 6.5M סיסמאות ב-SHA-1 בלי salt. נפרצו תוך שעות. ב-2016 התגלה שזה היה 117M. Encoding ≠ Encryption ≠ Hashing.
📖 מילון מושגים▶
Base64קידוד שהופך בינארי לטקסט ASCII. לא הצפנה.
AESAdvanced Encryption Standard — הצפנה סימטרית, 128/256 ביט.
RSAהצפנה אסימטרית — זוג מפתחות public/private.
SHA-256פונקציית hash שמייצרת 256 ביט. חד-כיוונית.
Digital Signatureחתימה עם private key שמאמתת זהות ושלמות.

Asymmetric = שני מפתחות (Public + Private). RSA, ECC.
Base64 הוא:
Base64 = encoding בלבד. כל אחד יכול לפענח. אף פעם אל תשתמש בזה להגנה.

אלגוריתמים — Hands-on
AES, RSA, ECC, SHA-256, Diffie-Hellman — ידיים על הקוד
ALGO25m▶
תיאוריה זה נחמד — אבל קריפטוגרפיה לומדים בידיים. בפרק הזה תכיר את האלגוריתמים העיקריים ותשחק איתם.
🔧AES = כספת אלקטרונית (מהירה, סימטרית). RSA = תיבת דואר (שני מפתחות). ECC = RSA מכווץ (אותה בטיחות, מפתח קטן יותר).
מה לומדים
AES — סימטרי, 128/256 ביט, מצבי ECB/CBC/GCM
RSA — אסימטרי, מפתחות 2048/4096 ביט
ECC — עקומים אליפטיים, 256 ביט = RSA 3072
SHA-256 / MD5 / bcrypt — hash functions
Diffie-Hellman — החלפת מפתחות על ערוץ פתוח
AES
Block size: 128 ביט. Key: 128/192/256. מצב GCM = הסטנדרט (הצפנה + אימות). ECB = שבור — הפינגווין המפורסם.
RSA
מבוסס על קושי פירוק מספרים ראשוניים. n = p × q. מפתח 2048 ביט = מינימום היום. איטי — משתמשים רק להחלפת מפתחות.
ECC
Elliptic Curve: y² = x³ + ax + b. מפתח 256 ביט של ECC = 3072 ביט RSA. Bitcoin משתמש ב-secp256k1.
HASH
MD5: 128 ביט, שבור — collisions תוך שניות. SHA-256: 256 ביט, בטוח. bcrypt: memory-hard, מושלם לסיסמאות.
DH
Diffie-Hellman: שני צדדים מייצרים מפתח משותף על ערוץ פתוח. g^a mod p, g^b mod p → (g^a)^b = (g^b)^a. בסיס של TLS.
💡PS3 Hack (2010): Sony השתמשה במספר אקראי קבוע (!) בחתימת ECDSA. תוקפים חילצו את ה-Private Key ופרצו את כל מערכת האבטחה.
📖 מילון מושגים▶
Block Cipherמצפין בלוקים קבועים (128 ביט ב-AES).
AEADAuthenticated Encryption with Associated Data — הצפנה + אימות.
NonceNumber used once — ערך חד-פעמי שמונע חזרה.
KDFKey Derivation Function — ממירה סיסמה למפתח (bcrypt, Argon2).

hash באורך 32 תווים hex — מה הסוג?
32 hex = 128 bits = MD5. SHA-1=40, SHA-256=64, bcrypt מתחיל ב-$2b$.
מה ה-mode המומלץ ב-AES היום?
AES-GCM = הצפנה + אימות. הסטנדרט של TLS 1.3. ECB שבור לחלוטין.

Crypto בעולם האמיתי
TLS, HTTPS, Certificates, JWT — איך הכל עובד ברשת
REAL25m▶
כל פעם שאתה גולש לאתר עם HTTPS — מתרחשת סדרת פעולות קריפטוגרפיות מורכבת תוך מילישניות. הפרק הזה מפרק את הקסם.
🤝TLS Handshake כמו פגישה ראשונה: שניכם מסכימים על שפה (cipher suite), מזדהים (certificates), ומייצרים סוד משותף (session key).
מה לומדים
TLS/SSL — ההיסטוריה וההבדלים (SSL שבור, TLS 1.3 בטוח)
HTTPS Handshake — ClientHello → ServerHello → Certificate → Key Exchange
Certificates — CA, chain of trust, X.509
JWT Tokens — header.payload.signature
01
ClientHello: הדפדפן שולח: גרסה, cipher suites נתמכים, random bytes. TLS_AES_256_GCM_SHA384
02
ServerHello: השרת בוחר cipher suite, שולח certificate + public key + random bytes.
03
Certificate Verify: הדפדפן מאמת את ה-certificate מול ה-CA (Let's Encrypt, DigiCert). Chain of trust עד ל-Root CA.
04
Key Exchange: Diffie-Hellman יוצר session key משותף. מכאן — AES-GCM מצפין הכל.
05
JWT: header.payload.signature. Header ו-Payload = Base64 (לא מוצפנים!). Signature = HMAC-SHA256 עם secret.
💡DigiNotar Hack (2011): CA הולנדי נפרץ. תוקפים הנפיקו certificates מזויפים ל-*.google.com. 300,000 איראנים רותחו ב-MITM. DigiNotar קרס.
📖 מילון מושגים▶
TLSTransport Layer Security — הפרוטוקול שמצפין תעבורת רשת.
CACertificate Authority — גוף שמנפיק תעודות דיגיטליות.
X.509הפורמט הסטנדרטי של certificates.
JWTJSON Web Token — טוקן אימות: header.payload.signature.

הדפדפן מתחיל עם ClientHello — שולח גרסה, cipher suites ו-random bytes.
ה-payload של JWT הוא:

Blockchain ומערכות קריפטו
Blocks, Hashes, Consensus, Smart Contracts
CHAIN25m▶
Blockchain הוא לא רק Bitcoin. זה מבנה נתונים קריפטוגרפי שמאפשר אמון בלי גורם מרכזי. כל בלוק מוצמד לקודם באמצעות hash.
📒Blockchain = פנקס חשבונות שמשוכפל אצל כולם. כל עמוד (בלוק) מכיל hash של העמוד הקודם. שינוי עמוד אחד → כל מה שאחריו נשבר.
מה לומדים
מה זה Blockchain — distributed ledger
Blocks: header, transactions, hash, nonce
Transactions: inputs, outputs, fees
Smart Contracts — Ethereum, Solidity
Gas fees — עלות חישוב על blockchain
Consensus: Proof of Work vs Proof of Stake
BLOCK
כל בלוק: prevHash + transactions + nonce + timestamp → SHA-256 → hash של הבלוק. שינוי בית אחד → hash שונה → השרשרת נשברת.
TX
Transaction: Input (מאיפה הכסף) + Output (לאן) + Fee (לכורה). חתום ב-ECDSA עם ה-Private Key של השולח.
POW
Proof of Work: מצא nonce שגורם ל-hash להתחיל ב-N אפסים. Bitcoin: ~10 דקות לבלוק. צורך אנרגיה אדירה.
POS
Proof of Stake: Ethereum 2.0. שם ETH כ-stake, נבחר אקראית לאמת. חוסך 99.9% אנרגיה.
SC
Smart Contract: קוד שרץ על blockchain. Solidity ב-Ethereum. כמו חוזה שמבצע את עצמו אוטומטית.
💡51% Attack: אם שחקן שולט ב-51% מכוח החישוב — יכול לכתוב מחדש היסטוריה. ב-2019 Ethereum Classic הותקף כך, $1.1M נגנב.
📖 מילון מושגים▶
Hashטביעת אצבע קריפטוגרפית של בלוק.
Nonceמספר שהכורה משנה עד שה-hash עומד בתנאי.
Gasיחידת מדידה לעלות חישוב ב-Ethereum.
Merkle Treeעץ hash שמאפשר אימות יעיל של transactions.
// CHECKPOINT
מה קורה כשמשנים transaction בבלוק ישן?
כל בלוק מכיל hash של הקודם. שינוי בלוק → hash משתנה → כל השרשרת שאחריו לא תקינה.
ההבדל העיקרי בין PoW ל-PoS?
PoW = כוח חישוב (אנרגיה). PoS = שם כסף כ-stake. Ethereum עבר ל-PoS ב-2022 וחסך 99.9% אנרגיה.

ארנקים דיגיטליים
Hot/Cold, Seed Phrases, MetaMask, Ledger
WALLET20m▶
הארנק שלך הוא לא באמת ארנק — הוא מחזיק מפתחות פרטיים. הכסף נמצא על ה-blockchain. מי שיש לו את המפתח — שולט בכסף.
🔐Private Key = מפתח לכספת. Public Key = מספר חשבון. Address = כתובת. Seed Phrase = גיבוי של כל המפתחות שלך.
מה לומדים
Hot Wallet vs Cold Wallet — מחובר לאינטרנט או לא
Custodial vs Non-custodial — מי מחזיק את המפתח?
Seed Phrases — BIP-39, 12/24 מילים
Private Key → Public Key → Address — שרשרת הגזירה
MetaMask, Ledger, Trezor — הכלים הנפוצים
HOT
Hot Wallet: MetaMask, Trust Wallet. מחובר לנט — נוח אבל פגיע. לשימוש יומי.
COLD
Cold Wallet: Ledger, Trezor. לא מחובר לנט — מפתח פרטי נשאר על המכשיר. $100+ = שמור על cold.
SEED
BIP-39: 12 או 24 מילים אנגליות = גיבוי של כל המפתחות. "abandon ability able..." → כתוב על נייר, לא בענן.
DERIVE
Private Key (256 ביט אקראיים) → Public Key (ECC secp256k1) → Address (hash + checksum). חד-כיווני — אי אפשר לחזור.
💡FTX Collapse (2022): מיליוני משתמשים שמו כסף ב-custodial wallet (FTX). כשקרס — הכל נעלם. "Not your keys, not your crypto."
📖 מילון מושגים▶
Seed Phrase12/24 מילים שמייצגות את כל המפתחות שלך.
BIP-39תקן של רשימת 2048 מילים ליצירת seed.
HD WalletHierarchical Deterministic — מייצר מפתחות ממקור אחד.
Custodialצד שלישי מחזיק את המפתח שלך (Coinbase, Binance).
// CHECKPOINT
מי שולט בקריפטו ב-custodial wallet?
Custodial = הפלטפורמה מחזיקה את ה-Private Key. אם היא קורסת (FTX) — אתה מפסיד הכל.
מה Seed Phrase?
BIP-39 Seed Phrase = 12/24 מילים שממנו נגזרים כל המפתחות. שמור על נייר — לעולם לא דיגיטלית.

Wallet Intelligence — מודיעין ארנקים
מעקב On-chain, Whale wallets, Arkham, Nansen
INTEL20m▶
Blockchain הוא ספר פתוח — כל transaction גלויה לכולם. מי שיודע לקרוא אותה יכול לעקוב אחרי כסף, לזהות לוויתנים, ולחזות מגמות.
🕵️On-chain analysis כמו חקירה פיננסית: כל ארנק משאיר עקבות. Cluster analysis מקשר ארנקים לבעלים. Whale alert = הגדולים זזים.
מה לומדים
Wallet tracking — מעקב on-chain
Wallet clusters — common spend analysis
Related owners identification
Whale wallets monitoring
כלים: Etherscan, Arkham, Nansen
TRACK
Etherscan: כתובת → כל ה-transactions, balances, tokens. etherscan.io/address/0x.... הכל פתוח.
CLUSTER
Wallet Clustering: אם שני ארנקים מופיעים כ-input באותה TX → סביר שאותו בעלים. "Common spend heuristic."
WHALE
Whale Tracking: ארנקים עם $10M+ נקראים whales. כשהם זזים → השוק זז. Whale Alert bot מדווח בזמן אמת.
TOOLS
Arkham Intelligence: מנוע AI שמקשר ארנקים לישויות אמיתיות. Nansen: labels ל-280M+ ארנקים. Chainalysis: לגופי אכיפה.
💡Colonial Pipeline (2021): FBI עקב אחרי $4.4M כופר ב-Bitcoin דרך on-chain analysis ושחזר $2.3M. Blockchain לא אנונימי — הוא pseudonymous.

On-chainנתונים שמופיעים על ה-blockchain עצמו.
Whaleארנק עם אחזקות גדולות ($10M+).
Mixer/Tumblerשירות שמערבב transactions כדי להקשות מעקב.
Pseudonymousלא אנונימי — הזהות מוסתרת אבל ניתנת לחשיפה.

זיהוי Scams
Rug Pulls, Honeypots, Phishing, Red Flags
SCAM20m▶
$7.7B נגנבו ב-crypto scams ב-2023 בלבד. Rug pulls, honeypots, phishing — התוקפים מתחדשים כל יום. הפרק הזה ילמד אותך לזהות דגלים אדומים לפני שאיבדת כסף.
🪤Rug Pull כמו חנות שנפתחת ביום שישי ונעלמת ביום ראשון — יוצרים token, מושכים נזילות, ובורחים עם הכסף.
מה לומדים
Rug Pulls — הסרת liquidity
Honeypot Contracts — אפשר לקנות, אי אפשר למכור
Phishing Wallets — approval exploits
Pump & Dump — ניפוח ומכירה
Red Flags checklist
RUG
Rug Pull: יוצר מוסיף liquidity → מוכר ל"משקיעים" → מוציא את כל ה-liquidity. Squid Game Token (2021): $3.3M נגנב.
HONEY
Honeypot: Smart contract שמאפשר קנייה אבל חוסם מכירה. הקוד מכיל require(msg.sender == owner) ב-transfer.
PHISH
Approval Exploit: מבקשים approve(MAX_INT) — נותן לתוקף גישה לכל הטוקנים שלך. תמיד תבדוק מה אתה חותם.
P&D
Pump & Dump: קבוצת Telegram/Discord מתאמת קנייה → המחיר עולה → הם מוכרים → אתה נשאר עם הפסד.
FLAGS
Red Flags: Anonymous team + Locked LP for only 30 days + No audit + Unlimited mint function + 99% held by one wallet.
💡Ronin Bridge Hack (2022): $625M נגנבו מ-Axie Infinity. Lazarus Group (צפון קוריאה) פרצו 5 מתוך 9 validators דרך phishing email מזויף של LinkedIn job offer.
📖 מילון מושגים▶
Rug Pullהסרת liquidity מ-DEX — המטבע מאבד ערך.
Honeypotחוזה שמאפשר קנייה אבל חוסם מכירה.
DEXDecentralized Exchange — בורסה ללא גורם מרכזי.
LPLiquidity Pool — מאגר נזילות ב-DEX.

התקפות קריפטו — Red Team
Replay, MITM, Heartbleed, JWT Attacks
ATTACK25m▶
הקריפטוגרפיה עצמה בדרך כלל לא נשברת — ה-implementation כן. Heartbleed, PS3 hack, Debian OpenSSL — כולם באגים בקוד, לא באלגוריתם.
⚔️הכספת חזקה — אבל מישהו השאיר את המפתח מתחת לשטיח. רוב ההתקפות מנצלות implementation flaws, לא חולשות מתמטיות.
מה לומדים
Replay Attacks — שליחה חוזרת של הודעה תקינה
MITM on TLS — Man in the Middle
Weak Randomness — PS3 hack, Debian OpenSSL
Key Leakage — Heartbleed
JWT Attacks — none algorithm, weak secret
REPLAY
Replay Attack: תוקף מקליט הודעה חתומה ושולח אותה שוב. פתרון: nonce או timestamp בתוך ההודעה.
MITM
MITM on TLS: תוקף מציג certificate מזויף. הגנה: Certificate Pinning, HSTS. DigiNotar hack = MITM על חצי מיליון איראנים.
RAND
Weak Randomness: Debian OpenSSL (2008) — באג הסיר אנטרופיה, רק 32,768 מפתחות אפשריים. PS3 ECDSA — nonce קבוע → Private Key חשוף.
BLEED
Heartbleed (2014): באג ב-OpenSSL heartbeat. שרת מחזיר 64KB מ-RAM — כולל מפתחות, סיסמאות, session tokens. 17% מהשרתים היו פגיעים.
JWT
JWT none attack: משנים alg: "none" בheader → שרת מקבל בלי signature. Weak secret: secret = "password" → Brute force תוך שניות.
💡SolarWinds (2020): תוקפים הזריקו backdoor ל-update של SolarWinds. 18,000 ארגונים התקינו — כולל Pentagon, Microsoft, FireEye. Code signing certificate היה תקין.

Foundations — Encryption vs Hashing vs Encoding
The difference between encoding, encryption and hashing
CORE20m▶
Three concepts everyone confuses: Encoding, Encryption, Hashing. They sound similar but do completely different things. Mix them up and you'll build broken systems.
📬Encoding = translating to another language (anyone can read). Encryption = locking in a safe (only key holder opens). Hashing = fingerprint (one-way, irreversible).
WHAT YOU'LL LEARN
Encoding (Base64, Hex) — format conversion, not protection
Encryption (AES, RSA) — encryption with a secret key
Hashing (SHA-256, MD5) — one-way function
Symmetric vs Asymmetric — one key or a pair
Public/Private keys — foundation of modern security
Digital signatures — proving identity
01
Encoding: Base64 converts binary to text. Hello → SGVsbG8=. Anyone can decode — this is NOT protection.
02
Symmetric Encryption: One key encrypts and decrypts. AES-256 — the standard. Like a safe with one key.
03
Asymmetric Encryption: Two keys — Public (encrypts) and Private (decrypts). RSA, ECC. Like a mailbox — anyone can insert, only you open.
04
Hashing: SHA-256 turns any input into 256 bits. One-way — can't reverse. Changing 1 bit → completely different hash.
05
Digital Signatures: Sign with Private Key, verify with Public Key. Proves message wasn't altered and sender is authentic.
💡Real-world: LinkedIn breach 2012 — 6.5M passwords in SHA-1 without salt. Cracked in hours. In 2016 it turned out to be 117M. Encoding ≠ Encryption ≠ Hashing.
📖 Glossary▶
Base64Encoding that converts binary to ASCII text. Not encryption.
AESAdvanced Encryption Standard — symmetric, 128/256 bit.
RSAAsymmetric encryption — public/private key pair.
SHA-256Hash function producing 256 bits. One-way.
Digital SignatureSigning with private key to verify identity and integrity.

Asymmetric = two keys (Public + Private). RSA, ECC.

Base64 = encoding only. Anyone can decode it. Never use for security.

Crypto Algorithms — Hands-on
AES, RSA, ECC, SHA-256, Diffie-Hellman
ALGO25m▶
Theory is nice — but cryptography is learned hands-on. This chapter covers the main algorithms and lets you play with them.
🔧AES = electronic safe (fast, symmetric). RSA = mailbox (two keys). ECC = compressed RSA (same security, smaller key).
WHAT YOU'LL LEARN
AES — symmetric, 128/256 bit, ECB/CBC/GCM modes
RSA — asymmetric, 2048/4096 bit keys
ECC — elliptic curves, 256 bit = RSA 3072
SHA-256 / MD5 / bcrypt — hash functions
Diffie-Hellman — key exchange over open channel
AES
Block size: 128 bits. Key: 128/192/256. GCM mode = standard (encryption + auth). ECB = broken — the famous penguin.
RSA
Based on difficulty of factoring primes. n = p × q. 2048-bit key = minimum today. Slow — only used for key exchange.
ECC
Elliptic Curve: y² = x³ + ax + b. 256-bit ECC = 3072-bit RSA. Bitcoin uses secp256k1.
HASH
MD5: 128 bits, broken — collisions in seconds. SHA-256: 256 bits, secure. bcrypt: memory-hard, perfect for passwords.
DH
Diffie-Hellman: Two parties create shared key over open channel. g^a mod p, g^b mod p → (g^a)^b = (g^b)^a. Foundation of TLS.
💡PS3 Hack (2010): Sony used a constant (!) random number in ECDSA signing. Attackers extracted the Private Key and broke the entire security system.
📖 Glossary▶
Block CipherEncrypts fixed blocks (128 bits in AES).
AEADAuthenticated Encryption with Associated Data.
NonceNumber used once — prevents repetition.
KDFKey Derivation Function — converts password to key.

TLS, HTTPS, Certificates, JWT
REAL25m▶
Every time you browse an HTTPS site — a complex series of cryptographic operations happens in milliseconds. This chapter breaks down the magic.
🤝TLS Handshake is like a first meeting: you agree on a language (cipher suite), identify yourselves (certificates), and create a shared secret (session key).
WHAT YOU'LL LEARN
TLS/SSL — history and differences (SSL broken, TLS 1.3 secure)
HTTPS Handshake — ClientHello → ServerHello → Certificate → Key Exchange
Certificates — CA, chain of trust, X.509
JWT Tokens — header.payload.signature
01
ClientHello: Browser sends: version, supported cipher suites, random bytes. TLS_AES_256_GCM_SHA384
02
ServerHello: Server chooses cipher suite, sends certificate + public key + random bytes.
03
Certificate Verify: Browser validates certificate against CA (Let's Encrypt, DigiCert). Chain of trust up to Root CA.
04
Key Exchange: Diffie-Hellman creates shared session key. From here — AES-GCM encrypts everything.
05
JWT: header.payload.signature. Header and Payload = Base64 (NOT encrypted!). Signature = HMAC-SHA256 with secret.

Blockchain & Crypto Systems
Blocks, Hashes, Consensus, Smart Contracts
CHAIN25m▶
Blockchain is not just Bitcoin. It's a cryptographic data structure that enables trust without a central authority. Each block is chained to the previous via hash.
📒Blockchain = ledger duplicated at everyone's. Each page (block) contains hash of the previous page. Change one page → everything after it breaks.
WHAT YOU'LL LEARN
What is Blockchain — distributed ledger
Blocks: header, transactions, hash, nonce
Transactions: inputs, outputs, fees
Smart Contracts — Ethereum, Solidity
Gas fees — computation cost on blockchain
Consensus: Proof of Work vs Proof of Stake
BLOCK
Each block: prevHash + transactions + nonce + timestamp → SHA-256 → block hash. Change one byte → hash changes → chain breaks.
TX
Transaction: Input (where money comes from) + Output (where it goes) + Fee (for miner). Signed with ECDSA using sender's Private Key.
POW
Proof of Work: Find nonce making hash start with N zeros. Bitcoin: ~10 min/block. Massive energy consumption.
POS
Proof of Stake: Ethereum 2.0. Stake ETH, randomly selected to validate. 99.9% energy savings.
SC
Smart Contract: Code running on blockchain. Solidity on Ethereum. Like a contract that executes itself automatically.
💡51% Attack: If a player controls 51% of computing power — they can rewrite history. In 2019, Ethereum Classic was attacked, $1.1M stolen.
📖 Glossary▶
HashCryptographic fingerprint of a block.
NonceNumber miner changes until hash meets condition.
GasUnit measuring computation cost on Ethereum.
Merkle TreeHash tree enabling efficient transaction verification.
// CHECKPOINT
What happens when you change a transaction in an old block?
Each block contains hash of the previous. Change a block → hash changes → entire chain after it is invalid.
Main difference between PoW and PoS?
PoW = computing power (energy). PoS = stake money. Ethereum switched to PoS in 2022, saving 99.9% energy.

Crypto Attacks — Red Team
Replay, MITM, Heartbleed, JWT Attacks
ATTACK25m▶
Cryptography itself usually doesn't break — the implementation does. Heartbleed, PS3 hack, Debian OpenSSL — all code bugs, not algorithm flaws.
⚔️The safe is strong — but someone left the key under the doormat. Most attacks exploit implementation flaws, not mathematical weaknesses.
WHAT YOU'LL LEARN
Replay Attacks — resending valid messages
MITM on TLS — Man in the Middle
Weak Randomness — PS3 hack, Debian OpenSSL
Key Leakage — Heartbleed
JWT Attacks — none algorithm, weak secret
REPLAY
Replay Attack: Attacker records signed message and sends it again. Fix: nonce or timestamp inside the message.
MITM
MITM on TLS: Attacker presents fake certificate. Defense: Certificate Pinning, HSTS. DigiNotar hack = MITM on half a million Iranians.
RAND
Weak Randomness: Debian OpenSSL (2008) — bug removed entropy, only 32,768 possible keys. PS3 ECDSA — constant nonce → Private Key exposed.
BLEED
Heartbleed (2014): OpenSSL heartbeat bug. Server returns 64KB from RAM — including keys, passwords, session tokens. 17% of servers were vulnerable.
JWT
JWT none attack: Change alg: "none" in header → server accepts without signature. Weak secret: secret = "password" → brute force in seconds.
💡SolarWinds (2020): Attackers injected backdoor into SolarWinds update. 18,000 organizations installed — Pentagon, Microsoft, FireEye. Code signing certificate was valid.

Replay AttackResending a legitimate message.
MITMMan in the Middle — attacker sits between two parties.
HeartbleedCVE-2014-0160 — RAM leak from OpenSSL.
Certificate PinningLocking specific certificate in application.
