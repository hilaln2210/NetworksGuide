מעבדה 6 — Troubleshooting: nginx לא עולה
systemctl · journalctl · ss · ports · logs
MEDIUM▶
תרגיל Debugging: nginx לא עולה. תשתמש ב-5 פקודות כדי לגלות מה הבעיה ולתקן אותה.
01התקן nginx: sudo apt install nginx -y
02הפעל והצג סטטוס: sudo systemctl start nginx && systemctl status nginx
03נגרום ל-nginx להיכשל: sudo systemctl start apache2 2>/dev/null || sudo apt install apache2 -y && sudo systemctl start apache2
04עצור nginx ונסה להפעיל שוב: sudo systemctl stop nginx && sudo systemctl start nginx
05בדוק את ה-Log: sudo journalctl -u nginx -n 20 — מה כתוב?
06מצא מי תפוס את פורט 80: sudo ss -tlnp | grep :80
07עצור apache2 ונסה שוב: sudo systemctl stop apache2 && sudo systemctl start nginx
✅ Troubleshooting Flow
05
ה-Log יראה: bind() to 0.0.0.0:80 failed (98: Address already in use). nginx רוצה להאזין לפורט 80 אבל מישהו כבר שם.
06
ss -tlnp | grep :80 יראה apache2 או process אחר. זה ה-Culprit.
07
אחרי שעוצרים apache2, פורט 80 פנוי ו-nginx עולה בהצלחה.
💡 5 שלבי Troubleshooting:
1. systemctl status SERVICE — Failed? Error code?
2. journalctl -u SERVICE -n 30 — מה כתוב ב-Log?
3. ss -tlnp — פורט תפוס?
4. df -h — דיסק מלא?
5. free -h — RAM מלא?
90% מהבעיות נפתרות בשלבים 1-3.
⚠️ תמיד sudo systemctl enable nginx לאחר הגדרה — אחרת nginx לא יעלה אוטומטית אחרי Reboot.
