✅ Answers — SQL Injection
02
' OR '1'='1: Query becomes: SELECT * FROM users WHERE id='' OR '1'='1'. Condition always true → returns all rows. Vulnerability: Input is inserted directly into the Query without sanitization.
04
UNION Attack: allows appending an additional Query to the original. Condition: same column count and types. information_schema = the built-in MySQL "database map" — contains names of all tables.
05
DVWA Users Table: The returned hashes are MD5. Can be cracked at CrackStation.net (Rainbow Tables). Admin: 5f4dcc3b5aa765d61d8327deb882cf99 = "password". This is why MD5 is forbidden for passwords.
06
High Level: DVWA adds a Token to the Request. SQL Security Level High level uses-Prepared Statements: $stmt = $db->prepare("SELECT * FROM users WHERE id=?"); $stmt->execute([$id]). Input becomes a parameter, not part of the Query.
⚠️ Run this exercise only on your own local DVWA. SQL Injection on systems without permission = criminal offense.
