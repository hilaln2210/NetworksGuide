❌ שרת מפרק 2 — בעיה
שתי מתודות חוסמות (blocking):

accept() — מקפיא עד לקוח חדש
recv() — מקפיא עד שמגיע מידע

בזמן שמחכים ל-recv מלקוח A — לקוח B לא יכול להתחבר!
בזמן שמחכים ל-accept — לקוח A לא יכול לשלוח!
✅ פתרון — select()
select() בודקת מי מוכן ללא חסימה:

"האם יש לקוח חדש שרוצה להתחבר?"
"האם לקוח A שלח מידע?"
"האם לקוח B מוכן לקבל?"

רק אחרי שיש מישהו מוכן — פועלים עליו.

import socket, select

server_socket = socket.socket()
server_socket.bind(('0.0.0.0', 23))
server_socket.listen(5)
open_client_sockets = []
messages_to_send = []

while True:
    rlist, wlist, xlist = select.select(
        [server_socket] + open_client_sockets,
        open_client_sockets,  # מי מוכן לקבל מידע
        []
    )

    for current_socket in rlist:
        if current_socket is server_socket:
            # לקוח חדש מתחבר
            new_socket, addr = server_socket.accept()
            open_client_sockets.append(new_socket)
        else:
            # לקוח קיים שלח מידע
            data = current_socket.recv(1024)
            if data == "":
                open_client_sockets.remove(current_socket)
            else:
                messages_to_send.append((current_socket, 'Hello, ' + data))

    send_waiting_messages(wlist)
🧠 לוגיקת הלולאה
1
select() חוסמת עד שמישהו מוכן. לא מבזבזת CPU.
2
אם server_socket ב-rlist → לקוח חדש מחכה → accept().
3
אם client_socket ב-rlist → לקוח שלח מידע → recv().
4
data == "" → לקוח ניתק → מסיר מהרשימה.
5
wlist = מי מוכן לקבל → שולחים הודעות מהתור אחרי שמעבדים קריאות.
send_waiting_messages — למה לא פשוט send()?

לא כל Socket מוכן לקבל מידע בכל רגע. send() על Socket לא מוכן = blocking = קיפאון. לכן: שומרים הודעות בתור, ושולחים רק ל-Sockets שהחזירו wlist. 

def send_waiting_messages(wlist):
    for message in messages_to_send:
        (client_socket, data) = message
        if client_socket in wlist:
            client_socket.send(data)
            messages_to_send.remove(message)
