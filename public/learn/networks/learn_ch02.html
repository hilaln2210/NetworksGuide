מושג יסוד
מה זה Socket?

Socket = נקודת קצה של חיבור. דמיינו צינור בין שתי תוכנות — מידע שנכנס מצד אחד יוצא בצד השני. כל קצה = Socket.

💡 אנלוגיה — בניין דירות
כתובת IP = כתובת הבניין (הרצל 1, תל אביב). פורט = מספר הדירה (דירה 23). Socket = תיבת הדואר הספציפית בדירה הזו.

🔁 0.0.0.0 לעומת 127.0.0.1

127.0.0.1 = Loopback — מתחבר לעצמך. לקוח ושרת על אותו מחשב.
API של Socket
כל הפונקציות
socket()
socket.socket()
יוצר אובייקט Socket חדש. נקודת ההתחלה — גם ללקוח וגם לשרת.
connect()
sock.connect(('ip', port))
לקוח בלבד. יוצר חיבור לשרת מרוחק לפי IP + Port.
bind()
sock.bind(('0.0.0.0', port))
שרת בלבד. קושר את ה-Socket לכתובת ופורט מקומי.
listen()
sock.listen(5)
שרת בלבד. מתחיל להאזין לחיבורים נכנסים.
accept()
(client_sock, addr) = sock.accept()
שרת בלבד. BLOCKING — מחכה עד שלקוח מתחבר. מחזיר Socket חדש לתקשורת.
⚠️ מקפיאה עד שלקוח מגיע!
send()
sock.send('Hello')
שולח מחרוזת (רצף בתים) לצד השני.
recv()
data = sock.recv(1024)
BLOCKING — מחכה עד שמגיע מידע. 1024 = מקסימום בתים לקריאה.
⚠️ מחרוזת ריקה = חיבור נסגר!
close()
sock.close()
סוגר את ה-Socket ומשחרר את הפורט. חשוב לקרוא בסוף!

server_socket = socket.socket()
server_socket.bind(('0.0.0.0', 8820))

(client_socket, client_address) = server_socket.accept()  # blocking!

שחרור משאבים
למה חשוב close()?
🔒 פורט תפוס — שגיאה נפוצה
אם התוכנה קרסה לפני close() — הפורט נשאר "תפוס". פתרון: שנה מספר פורט בצד הלקוח והשרת. אחרי זמן מה מערכת ההפעלה משחררת אוטומטית.
תרגיל 2.6 — פרוטוקול
שרת פקודות — TIME / NAME / RAND / EXIT

כל בקשה = בדיוק 4 bytes. תשובה — אורך משתנה. שרת קולט פקודה ומחזיר תגובה מתאימה:

TIME
מחזיר את השעה הנוכחית בשרת
import time → time.strftime(...)
NAME
מחזיר את שם השרת (מחרוזת קבועה)
"MyServer"
RAND
מחזיר מספר רנדומלי 1-10
import random → random.randint(1,10)
EXIT
מנתק את החיבור עם הלקוח
client_socket.close()
