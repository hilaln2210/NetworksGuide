🎯 תפקיד שכבת הרשת

שכבת הרשת (Layer 3) אחראית להעביר חבילות מידע (Packets) מישות אחת לישות אחרת — גם דרך מחשבים ורשתות רבות בדרך. כמו Waze שמחליטה על הנתיב, לא על אופן הנהיגה.

מה שכבת הרשת מספקת לLayer 4

מודל "ענן": שלח חבילה מכאן לכאן — ואני אדאג לדרך. Layer 4 לא יודע על נתבים, כבלים, WiFi.

מה שכבת הרשת מקבלת מLayer 2

שכבת הקו מטפלת בחיבור בין שתי ישויות סמוכות. Layer 3 לא אכפת איך — כבל, לוויין, יוני דואר.

📍 כתובות IP — מבנה ומשמעות

כתובת IPv4 = 4 bytes = 32 bits. כל byte = ערך 0-255. כתובת IP מחולקת לשני חלקים:

דוגמת "מחשביל'ה": 200.100.0.1 עם מסכה /16 — מזהה רשת=200.100, מזהה ישות=0.1

⚠ כתובת לא תקינה

192.168.275.2 — לא תקינה! 275 > 255, לא נכנס ב-byte.

ℹ כתובת IP היא לוגית, לא פיזית

IP אינה "צרובה" על כרטיס הרשת כמו MAC — ניתן לשנותה. ניתנת על ידי הרשת (DHCP) או ידנית.

🧮 Subnet Mask — מסיכת רשת

ה-Subnet Mask מגדיר כמה bits מהכתובת הם מזהה הרשת. ניתן לציין כ-/N (CIDR) או כ-255.x.x.x.

Subnet MaskCIDRמזהה רשתל-192.168.0.1 — מה ברשת?
255.0.0.0/8Byte ראשוןכל כתובת שמתחילה ב-192
255.255.0.0/162 Bytes ראשוניםכל כתובת שמתחילה ב-192.168
255.255.255.0/243 Bytes ראשוניםכל כתובת שמתחילה ב-192.168.0

💡 דוגמת בחינה מהפרק

192.160.0.1 ו-192.168.0.2 — אותה רשת ב-/8 (שניהם מתחילים ב-192), אבל רשתות שונות ב-/16 (192.160 ≠ 192.168)!

🔄 NAT — Network Address Translation

בסוף שנות ה-80 נגמרו כתובות IPv4. הפתרון: NAT. כל הרשת הפרטית משתמשת בכתובת ציבורית אחת של הנתב.

→ יציאה לאינטרנט

מחשב (192.168.0.1) שולח ל-Google.
נתב מחליף src: 192.168.0.1 → 1.1.1.1.
Google רואה רק 1.1.1.1.

← תשובה חזרה

Google שולח ל-1.1.1.1 (הנתב).
נתב מחליף dst: 1.1.1.1 → 192.168.0.1.
המחשב מקבל כאילו Google ענה ישירות.

❓ שאלה מהפרק

כשמחשב א' ומחשב ב' פונים שניהם ל-Google — איך הנתב יודע למי להחזיר? התשובה: ה-NAT table שומר מיפוי של פורטים. לא נפורט בספר זה.

🗺 טבלת ניתוב (Routing Table)

לכל נתב (ומחשב) יש טבלת ניתוב. הנתב בודק מלמטה למעלה — מהכלל הספציפי לגנרי.

#Network DestinationNetmaskInterface
10.0.0.00.0.0.03.3.3.3 (R4 — Default)

חבילה ל-100.200.5.8: בודק שורה 3 → מזהה רשת 100.200 ✓ → שולח ל-R3 (2.2.2.2)

💡 Default Gateway = שורת 0.0.0.0/0

כל חבילה שלא התאימה לאף שורה — הולכת ל-Default Gateway. זהו הנתב שמחבר אתנו לאינטרנט.

💻 route print — טבלת ניתוב של המחשב

בשורת פקודה: route print. רואים Default Gateway, כתובות לממשקים שונים (כרטיס רשת + loopback).

📡 ICMP — Internet Control Message Protocol

פרוטוקול לאבחון רשת. Header קבוע של 8 bytes. שני כלים מרכזיים שמשתמשים בו:

Ping = Echo Request/Reply

Type=8 (Echo Request), Type=0 (Echo Reply). Code=0 תמיד. הגוף מועתק בתשובה. Identifier ו-Sequence Number מאפשרים לזהות תגובות.

# Scapy — ping
pkt = IP(dst='www.google.com')/ICMP()
resp = sr1(pkt)

ICMP Type=11 — Time Exceeded

נשלח כשנתב מוריד TTL ל-0. מכיל העתק של החבילה המקורית. זהו הבסיס של Traceroute!

# Time Exceeded → כתובת הנתב
hop_ip = resp[IP].src

⏱ TTL ו-Traceroute — כיצד זה עובד

TTL (Time To Live) = שדה ב-IP Header (1 byte, 0-255). כל נתב מוריד TTL ב-1. כש-TTL=0 — הנתב זורק את החבילה ושולח ICMP Time Exceeded חזרה.

מטרת TTL: מניעת לולאות אינסופיות. חבילה "לכודה" בין נתבים תיזרק.

💡 כיצד Traceroute בנוי — הטריק

שלח פקטה עם TTL=1 → הנתב הראשון שולח Time Exceeded → מגלים כתובתו.
שלח TTL=2 → נתב שני זורק → מגלים כתובתו.
וכך הלאה עד שמגיעים ליעד. 

# Scapy — Traceroute ידני
for ttl in range(1, 30):
    pkt = IP(dst='www.google.com', ttl=ttl) / ICMP()
    resp = sr1(pkt, timeout=2)
    if resp:
        print(resp[IP].src)
        if resp[IP].src == IP(dst='www.google.com').dst:
            break

📧 DHCP — קבלת כתובת IP דינמית

DHCP (Dynamic Host Configuration Protocol) מקצה כתובת IP לכרטיס רשת שהתחבר לרשת. תהליך בן 4 הודעות:

מחשב →
DHCP Discover — "אין לי IP, מישהו יעזור?"
נשלח ל-Broadcast (255.255.255.255), src=0.0.0.0
שרת DHCP →
DHCP Offer — "הנה הצעה: IP=192.168.1.5, Lease=24h"
גם נשלח ל-Broadcast (הלקוח עדיין בלי IP). כולל Subnet Mask, Gateway, DNS
מחשב →
DHCP Request — "אני בוחר בהצעה שלך"
Broadcast — גם שאר שרתי DHCP יודעים שלא נבחרו
שרת DHCP →
DHCP ACK — "מאושר! הכתובת שלך עד..."
המחשב יכול להתחיל לתקשר. קיבל: IP, Mask, Gateway, DNS

🔧 ipconfig /release ו-/renew

ipconfig /release = שחרר כתובת IP (מחשב שולח DHCP Release).
ipconfig /renew = בקש כתובת חדשה (מתחיל תהליך Discover מחדש).

❓ למה DHCP מעל UDP ולא TCP?

שתי סיבות: (1) שאלה-תשובה קצרה — אם אבדה שולחים שוב. (2) הודעות נשלחות ל-Broadcast — TCP לא תומך ב-Broadcast.
