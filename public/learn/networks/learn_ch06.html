📡 פרק 6 — שכבת התעבורה

Ports · TCP vs UDP · Three-Way Handshake · Seq/ACK Numbers · Checksum · netstat

🎯 מה תפקידה של שכבת התעבורה?

שכבת התעבורה (Layer 4) אחראית להעברת מידע מתוכנה לתוכנה, לא רק ממחשב למחשב. יש לה שתי מטרות:

📦 1. ריבוב אפליקציות (תמיד)

מאפשרת להפנות בקשות לשרותים שונים על אותה IP. כמה תוכנות יכולות לרוץ על אותו מחשב ולהאזין כל אחת על פורט שונה.

🏢 IP = כתובת הבניין
🚪 Port = מספר הדירה

🛡 2. העברה אמינה (אופציונלי)

TCP מבטיח שהמידע יגיע ובסדר הנכון. UDP לא מבטיח — אבל מהיר יותר.

לא תמיד צריך אמינות — לפעמים מהירות חשובה יותר (Skype, DNS)

🔑 פורטים (Ports)

פורט הוא מספר בין 0 ל-65,535 שמזהה תוכנה ספציפית על מחשב.

Well-Known Ports (0–1023)

מוקצים בידי IANA. דורשים הרשאות admin.

פורטשירות
80HTTP
443HTTPS
22SSH

פורט פתוח vs סגור

פתוח = תוכנה מאזינה, מוכנה לקבל חיבורים.
סגור = אין תוכנה מאזינה.

הכלי netstat מציג פורטים פתוחים:

netstat -a
# -a = כולל LISTENING
# LISTENING = מחכה לחיבור
# ESTABLISHED = חיבור פעיל

🌀 TCP לעומת UDP

TCP — מבוסס קישור

  • מבטיח הגעת מידע
  • מבטיח סדר נכון
  • צריך להרים קישור (3-way handshake)
  • Overhead גבוה
  • כמו שיחת טלפון: חייגים, מדברים, מנתקים

מתי: הורדת קבצים, HTTP, תמונות, מייל

UDP — לא מבוסס קישור

  • לא מבטיח הגעה
  • לא מבטיח סדר
  • אין handshake — שולחים ישר
  • Overhead נמוך, מהיר
  • כמו שליחת מכתב בדואר רגיל

מתי: DNS, VoIP, סטרימינג, משחקים

מקרהפרוטוקולסיבה
הורדת קובץ גדולTCPכל הנתונים חייבים להגיע בסדר הנכון
DNS queryUDPשאלה+תשובה קצרה, לא שווה להרים TCP
Skype / VoIPUDPמהירות חשובה מאמינות, packet אבוד = סנטימטר שיחה
העלאת תמונותTCPכל pixel חייב להגיע
HTTPTCPדף אינטרנט שלם חייב להגיע

💬 שאלת חשיבה מהפרק

למה בכלל צריך UDP אם שכבת הרשת גם היא לא אמינה? תשובה: (1) פורטים — UDP מוסיף פורטים שאי אפשר בלעדיהם. (2) מודל השכבות — מפתח שכבת אפליקציה לא צריך להכיר Layer 3.

📦 UDP Header — 8 בתים בלבד

שדות UDP Header

שדהגודלתפקיד

Destination Port2Bפורט היעד

Socket UDP בקוד Python

# לקוח UDP
sock = socket.socket(
    socket.AF_INET,
    socket.SOCK_DGRAM  # UDP!
)
sock.sendto('Hello', ('1.2.3.4', 8821))
data, addr = sock.recvfrom(1024)

# שרת UDP — אין listen/accept!
sock.bind(('0.0.0.0', 8821))
data, addr = sock.recvfrom(1024)
sock.sendto('Hi', addr)

💡 Checksum — מה זה ואיך עובד?

Checksum = חישוב על המידע שנשלח. השולח שולח את הערך. המקבל מחשב מחדש ומשווה. אם שונה — שגיאה, חבילה נזרקת. בUDP זה אופציונלי (ניתן לשים 0).

🔢 TCP — Sequence Numbers ו-ACK

TCP נותן מספר סידורי לכל בית (byte) בחיבור — לא לכל חבילה. כך אפשר לדעת מה חסר ומה הגיע בסדר לא נכון.

📋 איך זה עובד — דוגמה מהפרק

שולחים "Hello cool network!" החל מ-Seq=100:

לקוח → שרת
Seq=100 "Hello " len=6
שרת → לקוח
ACK=106 "קיבלתי עד byte 105"
לקוח → שרת
Seq=106 "cool" len=4
שרת → לקוח
ACK=110 "קיבלתי עד byte 109"

💡 כלל חישוב ACK

ACK = Seq_last + len(data)
ACK מציין את הבית הבא שמצופה — כלומר כל מה שלפניו הגיע תקין.

⚠ אם חבילה אבדה

הלקוח מחכה ACK. אחרי timeout — שולח שוב. השרת יסדר את החבילות לפי Seq Number גם אם הגיעו בסדר לא נכון.

🤝 Three-Way Handshake — הקמת קישור TCP

כל חיבור TCP מתחיל ב-3 חבילות. בדוגמה: לקוח ISN=123, שרת ISN=555.

Client
→
SYN   Seq=123, ACK=0 (כבוי)
הלקוח: "אני רוצה להתחבר, ה-ISN שלי הוא 123"
←
SYN + ACK   Seq=555, ACK=124
השרת: "הסכמתי, ISN שלי 555. קיבלתי את הSYN שלך (ACK=123+1=124)"
→
ACK   Seq=124, ACK=556
הלקוח: "קיבלתי את הSYN+ACK שלך (ACK=555+1=556). הקישור קם!"
Server

🧮 כלל חישוב ACK ב-Handshake

SYN נחשב כ-byte אחד (גם שלא נשלח מידע). לכן:
● SYN שלח Seq=123 ← ACK של השרת = 123+1 = 124
● SYN+ACK שלח Seq=555 ← ACK של הלקוח = 555+1 = 556

❓ למה ISN רנדומלי?

אם ISN=0 תמיד — חיבורים ישנים שבדרך יכולים לבלבל חיבורים חדשים. ISN רנדומלי מונע התנגשויות בין חיבורים.

🔍 Port Scanning עם SYN

אם שולחים SYN לפורט ומקבלים SYN+ACK — פורט פתוח. אם מקבלים RST — פורט סגור. אם לא מגיע כלום — חסום על ידי Firewall. זהו עיקרון Port Scanner (תרגיל 6.19).

💻 UDP ו-TCP ב-Scapy

שאילתת DNS ב-Scapy (תרגיל 6.10)

dns_pkt = (IP(dst='8.8.8.8') /
    UDP(sport=24601, dport=53) /
    DNS(qdcount=1, rd=1) /
    DNSQR(qname='www.google.com'))

# שלח + קבל תשובה
resp = sr1(dns_pkt)
resp.show()

TCP SYN ב-Scapy (תרגיל 6.18)

# שלב 1: SYN
syn = TCP(dport=80, seq=123, flags='S')
pkt = IP(dst='www.google.com') / syn
resp = sr1(pkt)  # מקבל SYN+ACK

# שלב 2: ACK
ack = TCP(
    dport=80,
    seq=resp[TCP].ack,
    ack=resp[TCP].seq+1,
    flags='A'
)
send(IP(dst='www.google.com') / ack)
