📡 HTTP – איך זה עובד?

HTTP הוא פרוטוקול בקשה-תגובה: הלקוח (דפדפן) שולח Request, השרת מחזיר Response. כל תקשורת HTTP כוללת שורת בקשה/תגובה + Headers + Data (תוכן).

דפדפן
──── GET /page.html ────→
שרת
←── 200 OK + HTML ────
דפדפן

📌 מה ראינו ב-Wireshark (תרגיל 4.1)

הסנפנו גלישה ל-info.cern.ch/hypertext/WWW/TheProject.html עם פילטר http. ראינו שתי שורות: פקטת הבקשה מהלקוח, ופקטת התגובה מהשרת.

📝 מבנה בקשת GET

GET /hypertext/WWW/TheProject.html HTTP/1.1
Host: info.cern.ch
Accept: text/html
Accept-Language: he
Connection: keep-alive
          <-- שורה ריקה = סיום Headers
          <-- GET אין Body בבקשת

📋 שורת הבקשה

METHOD + URL + גרסת פרוטוקול
GET /page.html HTTP/1.1

📋 Headers

שדות מידע נוספים. פורמט: שם: ערך
מסתיימים בשורה ריקה (\r\n\r\n)

📋 Body (Data)

תוכן הבקשה. ב-GET ריק. ב-POST כולל את הנתונים שנשלחים.

ℹ הפרדה בין שורות

ב-HTTP משתמשים ב-\r\n (carriage return + line feed) להפרדת שורות. שורה ריקה (\r\n\r\n) = סיום Headers תחילת Body.

📩 מבנה תגובת HTTP

HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Content-Length: 1234
Server: Apache
          <-- שורה ריקה
<html>...תוכן דף האינטרנט...</html>

שורת התגובה מכילה 3 שדות: גרסת פרוטוקול · קוד תגובה · תיאור.

💡 לוגיקת הלקוח

הלקוח קודם בודק קוד תגובה ← אם 200 בודק Content-Type ← מציג את ה-Body. אם 404 מציג שגיאה. אם 301/302 שולח בקשה חדשה לכתובת שב-Location Header.

🔄 GET לעומת POST

GET – בקשת קבלת מידע

• מידע עובר ב-URL כפרמטרים
• מקסימום ~2000 תווים
• גלוי לעין בדפדפן
• ניתן לשמור כ-Bookmark
• לא לשינוי נתונים בשרת

GET /search?q=python&lang=he HTTP/1.1

POST – שליחת נתונים

• מידע עובר ב-Body
• גודל לא מוגבל (תמונות, קבצים)
• לא גלוי ב-URL
• לא ניתן לשמור כ-Bookmark
• לשינוי/יצירת נתונים

POST /upload HTTP/1.1
Content-Length: 45123
<-- שורה ריקה -->
[תוכן התמונה בינארי...]

📸 דוגמה מהפרק: העלאת תמונה ל-Facebook

תמונה בטלפון = 1MB+. URL מקסימלי = 2000 תווים. לא ניתן להעביר בGET. לכן Facebook משתמש ב-POST ששולח את התמונה ב-Body.

🔍 דוגמה: Google Search

חיפוש ב-Google = בקשת GET עם פרמטר q:
GET www.google.com/search?q=python HTTP/1.1
חיפוש מרובה פרמטרים מופרד ב-&:
GET twitter.com/search?q=obama&mode=users HTTP/1.1

🔗 URL – כתובת משאב

http://www.site.com/folder_a/folder_b/file.txt
חלקדוגמהמשמעות
Protocol (Scheme)http://פרוטוקול התקשורת
Hostwww.site.comשם הדומיין / IP השרת
Path/folder_a/folder_b/file.txtנתיב לקובץ ב-root directory
Query String?q=python&lang=heפרמטרים לבקשת GET (אחרי ?)

📁 Root Directory

תיקיית הבסיס שהשרת מגיש ממנה קבצים. פנייה ל-/folder/file.txt מתורגמת ל-C:\wwwroot\folder\file.txt. פנייה ל-/ = עמוד הבית.

🚀 תרגיל 4.4 – שרת HTTP בסיסי

מה שהשרת עושה בכל בקשה:

# 1. קבל חיבור מלקוח
(client_socket, addr) = server_socket.accept()

# 2. קרא את הבקשה
request = client_socket.recv(1024)

# 3. פרסר את שורת הבקשה
# "GET /index.html HTTP/1.1\r\nHost: ..."
method, url, protocol = request.split(' ')[:3]

# 4. קרא את הקובץ
with open(root_dir + url, 'rb') as f:
    data = f.read()

# 5. שלח תגובה
response = "HTTP/1.0 200 OK\r\n"
response += "Content-Length: " + str(len(data)) + "\r\n"
response += "\r\n"
client_socket.send(response)
client_socket.send(data)

🧮 תרגיל 4.6 – GET Parameters

שרת שמחשב next-number לפי פרמטר:

# בקשה: GET /calculate-next?num=4 HTTP/1.1
# מחלץ פרמטרים מה-URL
url = "/calculate-next?num=4"
path, params = url.split('?')
# path = "/calculate-next", params = "num=4"
key, val = params.split('=')
result = str(int(val) + 1)
# result = "5"

# בקשה: GET /calculate-area?height=3&width=4
# תשובה: "6.0"

⚠ שגיאות נפוצות בתרגיל שרת HTTP

• לא לשכוח \r\n בין Headers
• שורה ריקה (\r\n\r\n) חייבת להפריד Headers מ-Body
• קריאת קבצים ב-binary mode ('rb') לתמונות ו-JS
• Windows: להחליף / ב-\ בנתיבי קבצים
