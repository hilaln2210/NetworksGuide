Encoding vs Encryption vs Hashing
Base64, Encryption, Hashing — three completely different things
CORE⏱ 25m▶
Many people confuse Encoding, Encryption, and Hashing. It's like the difference between putting a letter in an envelope, locking it in a safe, or shredding it.
Encoding
HELLO → SEVMTE8= (Base64). Anyone can reverse it. This is not security — it's format.
Encryption
HELLO + key → x8#kL. Only someone with the key can reverse it. Two-way with a secret.
Hashing
HELLO → 2cf24dba (SHA-256). One-way. No going back. Used for: passwords, verification.
💡Base64 ≠ Encryption! If you see eyJhbGciOiJIUzI1NiJ9 in a JWT — that's Base64. Anyone can decode it. Zero security.
📖 Glossary▶
Base64Binary→text encoding. 3 bytes → 4 characters. Common in email, JWT.
HexBase-16 representation. Each byte = 2 hex digits (00-FF).
HashOne-way function. Any change → completely different hash.
