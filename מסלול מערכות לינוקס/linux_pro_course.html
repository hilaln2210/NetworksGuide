EASY▶
"הכל קובץ" — זה לא רק סיסמה, זו ארכיטקטורה. תוכיח לעצמך שהיא נכונה.
01רוץ ls -lai /proc/self/ — מה ה-inode number של . ו-..?
02צור קובץ, צור Hard Link אליו, בדוק ש-inode זהה: stat file1 file2
03מחק את הקובץ המקורי — האם תוכן ה-Link נשמר? למה?
04חקור את /proc/$(pidof bash)/fd/ — מה הקבצים הפתוחים?
05מצא את כל הקבצים הגדולים מ-100MB: find / -size +100M -type f 2>/dev/null
06שאלה: Directory ל-Filesystem Root מרוקן — כמה Inodes נשרפו?
✅ תשובות
02
Hard Links חולקים inode. ln file1 file2 → stat יראה Inode: 12345678 זהה לשניהם. Link count = 2.
03
כן, תוכן נשמר! rm file1 רק מוחק את ה-Directory Entry. inode נמחק רק כש-Link Count יורד ל-0 וכל File Descriptors נסגרים.
04
/proc/self/fd/ מכיל Symlinks לכל FD פתוח: fd/0 = stdin, fd/1 = stdout, fd/2 = stderr. ls -la /proc/self/fd/ יראה גם sockets.
06
Directory ריק עדיין צורך 1 inode. Filesystem מלא ב-inodes (לא ב-space) = לא ניתן ליצור קבצים חדשים. בדוק: df -i
💡 /proc/meminfo, /proc/cpuinfo, /proc/net/dev — כל ה"קבצים" האלה נוצרים בזמן אמת על ידי ה-Kernel. קראת אותם? ה-Kernel כתב אותם באותה שניה.

מעבדה 2 — Bash Script ברמת Production
set -euo pipefail · Traps · Logging · Error Handling · Idempotency
EASY▶
כתוב Bash Script שגם מהנדס Senior ב-Google ייתן לו Approve בCode Review.
01צור script עם set -euo pipefail + IFS=$'\n\t' בתחילה. למה?
02הוסף Trap: trap 'cleanup' EXIT ERR — cleanup מנקה temp files ומדפיס סיבת כשל
03כתוב log() function: [2024-01-15 14:32:01] [INFO] Starting backup...
04גרום ל-Script להיות Idempotent — הרצה פעמיים = אותה תוצאה
05הוסף Lockfile: מניעת ריצה מקבילה של אותו script
✅ Production-Grade Bash Template
backup.sh
#!/usr/bin/env bash
set -euo pipefail
IFS=$'\n\t'

# --- Constants ---
readonly LOCK=/tmp/backup.lock
readonly LOG=/var/log/backup.log

# --- Logging ---
log() { printf '[%(%F %T)T] [INFO] %s\n' -1 "$*" | tee -a "$LOG"; }
die() { printf '[%(%F %T)T] [ERROR] %s\n' -1 "$*" >&2; exit 1; }

# --- Cleanup ---
cleanup() {
  local exit_code=$?
  rm -f "$LOCK"
  [[ $exit_code -ne 0 ]] && die "Script failed (exit $exit_code)"
}
trap cleanup EXIT

# --- Lockfile ---
if ! ( set -o noclobber; echo $$ > "$LOCK" ) 2>/dev/null; then
  die "Already running (PID $(cat $LOCK))"
fi

log "Backup started"
💡
set -u — משתנה לא מוגדר = שגיאה מיידית. מנע את GitLab DB Bug. pipefail — false | true = כשל בלי pipefail. עם pipefail = script נכשל. IFS=$'\n\t' — מניעת Word Splitting בעת עבודה עם Paths עם רווחים.
⚠️ Lockfile עם echo $$ > file לא Atomic — Race Condition. set -o noclobber + Subshell = Atomic. זה ה-Pattern הנכון.
